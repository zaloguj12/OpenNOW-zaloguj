# Android Client Clean Rebuild - Quick Start

## 📋 The Problem

The Android branch is **205 commits behind** upstream with significant code divergence:
- Android-specific changes mixed throughout `src/main/`, `src/renderer/`, build configs
- Merging upstream requires manual conflict resolution + recoding
- Every new version release requires cherry-picking and refactoring

## 🎯 The Solution

**Rebuild Android as an optional platform layer, not a fork fork divergence.**

Core principle: **Zero Android code in src/shared/, minimal changes to src/main/ and src/renderer/.**

## 🏗️ Three-Layer Architecture

```
┌─ SHARED (src/shared/)          ← Platform-agnostic (UNTOUCHABLE)
├─ PLATFORM LAYER (src/android/) ← Android-specific code (ISOLATED)
└─ PLATFORM TARGETS              ← Electron vs Android vs Web
```

## 📊 Before vs. After

| Metric | Before | After |
|--------|--------|-------|
| Merge conflicts | ❌ Frequent | ✓ Rare (<3) |
| Divergence | 205 commits behind | 0 (catchable) |
| Files modified | 10+ scattered | 2-3 consolidated |
| Merge effort | Days + recoding | Hours + git rebase |

## 🚀 Implementation Phases

### Phase 1: Foundation (3 days)
- Create `src/platform.ts` for runtime detection
- Extract auth to shared contract + platform-specific implementations
- Organize Android code in `src/android/`
- **Result**: Electron still works, new structure in place

### Phase 2: Build Config (3 days)
- Create `vite.config.web.ts` for Capacitor builds
- Add `capacitor.config.json`
- Update build scripts
- **Result**: Can build both `npm run dist` (Electron) and `npm run build:web` (Android)

### Phase 3: Integration (3 days)
- Platform detection in `src/main/index.ts`
- Conditional bridging in `src/preload/index.ts`
- Capacitor initialization in `src/renderer/App.tsx`
- **Result**: App works on both platforms

### Phase 4: Verification (1 day)
- All tests pass
- Electron build identical to before
- **Test merge upstream** ← The critical proof
- **Result**: Clean merge confirmed

### Phase 5: Capacitor/Native (4 days)
- Create `GfnPlugin.kt` for OAuth + token storage
- Wire Android build pipeline
- Test on device


## 🔑 Key Design Principles

1. **`src/shared/` is untouchable** - Zero platform knowledge
2. **Android code in one place** - `src/android/` directory
3. **Configuration-driven** - Use config files, not code hardcoding
4. **Minimal changes to existing code** - Backward compatible
5. **Clean contracts** - Shared interfaces, platform-specific implementations

## 📌 Why This Works for Merges

When upstream releases v0.3.9:
1. Checkout `main`, pull latest
2. Rebase `android/main` on top: `git rebase main`
3. **Result**: 0 conflicts (99% of the time)
   - Why? Upstream changes are in `src/shared/`, `src/main/`, `src/renderer/`
   - Android changes are in `src/android/` and config files
   - No overlap = no conflicts

## 🤔 FAQ

**Q: Will this break the Electron build?**
A: No. Electron code path remains unchanged. All Android changes are isolated.

**Q: Do I need to rewrite all the business logic?**
A: No. `src/shared/` (auth, WebRTC, etc.) stays the same. Only auth **initialization** moves to platform-specific modules.

**Q: Can Android and Electron coexist?**
A: Yes. The app detects platform at startup and routes to appropriate implementations.

**Q: What if upstream changes a file in src/android/?**
A: Extremely unlikely. Upstream doesn't know about `src/android/`. If it happens, resolve manually (rare).

**Q: How long does this take?**
A: ~2 weeks for implementation + testing. Ongoing maintenance: minutes per upstream release.

**Q: Can I do this incrementally?**
A: Yes. After Phase 2, you have a working split build. Android developers can work on Capacitor in parallel.
