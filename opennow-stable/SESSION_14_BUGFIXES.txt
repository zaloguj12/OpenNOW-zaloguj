DEVICE-SPECIFIC NOTES
----------------------
HyperOS (Xiaomi's Android skin) handles edge-to-edge and system bar
insets differently from stock Android. Specifically:
- windowTranslucentStatus/Navigation flags can conflict with the modern
  WindowCompat API and cause env() insets to return 0.
- The correct approach for HyperOS is to use WindowCompat.setDecorFitsSystemWindows
  (already done in MainActivity) PLUS set bar colors to transparent BOTH in
  the theme XML and at runtime in onCreate().
- windowLayoutInDisplayCutoutMode=shortEdges is also needed so the notch
  area is handed to the app correctly.
- The narrow screen (~393dp) is why the library toolbar was overflowing --
  icon + title + search + count was too wide.

--- SAFE AREA FIXES ---

MainActivity.kt:
  - WindowCompat.setDecorFitsSystemWindows(window, false)  [was already there]
  - Added: window.statusBarColor = Color.TRANSPARENT
  - Added: window.navigationBarColor = Color.TRANSPARENT
  These two lines are required for HyperOS to populate env() insets correctly.

styles.xml (AppTheme.NoActionBar):
  - Removed: windowTranslucentStatus (conflicts with WindowCompat API)
  - Removed: windowTranslucentNavigation (conflicts with WindowCompat API)
  - Added: android:statusBarColor = transparent
  - Added: android:navigationBarColor = transparent
  - Added: android:windowLayoutInDisplayCutoutMode = shortEdges

styles.css (.app-container):
  - Added: padding-top: env(safe-area-inset-top, 0px)
  - Pushes all content below the status bar.

styles.css (.navbar):
  - Changed height from var(--navbar-h)
    to calc(var(--navbar-h) + env(safe-area-inset-bottom, 0px))
  - padding-bottom: env(safe-area-inset-bottom, 0px)  [was already there]
  - The height grows to accommodate the gesture strip, padding pushes tabs up.

styles.css (.main-content):
  - margin-bottom: calc(var(--navbar-h) + env(safe-area-inset-bottom, 0px))
  - Keeps content above the navbar including the safe area.
