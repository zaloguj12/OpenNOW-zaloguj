PROGRESS LOG - OpenNOW Android
Last updated: 2026-02-24

== CURRENT STATUS ==
Stream connects successfully end-to-end on Android.
Public games, library games, session creation, signaling, and WebRTC
all run in the WebView using browser-native fetch/WebSocket/WebRTC.
The only Kotlin plugin calls remaining are: auth (login/logout/getAuthSession),
settings (getSettings/setSetting), and the session conflict dialog.

The last known issue was the app kicking back to the library immediately
after connecting. This was fixed -- see SESSION 3 below.

README.md has been rewritten to Android-only with full build instructions.

== WHAT WORKS ==
- Login (NVIDIA OAuth PKCE via LoginActivity WebView)
- Token persistence and refresh (EncryptedSharedPreferences)
- Main games list (GraphQL panels API)
- Library games list (GraphQL panels API)
- Public games list (static JSON)
- Switching between game sources (main / library / public)
- Resume session button (getActiveSessions now parses correctly)
- Session creation and polling (full cloudmatch.ts request body)
- Signaling WebSocket (BrowserSignalingClient, runs in WebView)
- WebRTC offer/answer/ICE exchange
- Game video and audio streaming
- Touch input (TouchInputHandler)
- On-screen virtual gamepad (TouchGamepad)
- Settings (stored in SharedPreferences via GfnPlugin)
- Session stop

== WHAT IS NOT TESTED / UNKNOWN ==
- Microphone input on Android
- Release/signed APK build
- Devices below API 30

-- SESSION 3 (major) --
Problem: black screen on public games switch, session timeout, no resume button.
Root cause: Kotlin reimplementations of session/games API were incomplete and
used wrong request bodies. Could not be kept in sync with server behavior.

Fix: moved session management and games API entirely into the JS/WebView layer
by importing cloudmatch.ts and games.ts directly into the renderer bundle.

Changes made:
  electron.vite.config.ts
    - Added @main alias pointing to src/main/
    - Added node:crypto alias pointing to cryptoShim.ts

  src/renderer/src/platform/cryptoShim.ts  [NEW]
    - Browser shim for node:crypto
    - Exports randomUUID() using globalThis.crypto.randomUUID()

  src/renderer/src/platform/api.ts
    - fetchMainGames / fetchLibraryGames / fetchPublicGames: now call
      games.ts functions directly (browser fetch, no Kotlin)
    - createSession / pollSession / stopSession / getActiveSessions /
      claimSession: now call cloudmatch.ts functions directly (browser fetch)
    - connectSignaling / disconnectSignaling / sendAnswer / sendIceCandidate:
      now use BrowserSignalingClient (browser WebSocket, no Kotlin)
    - Removed ensureSignalingEvents() and Capacitor plugin listener

  src/renderer/src/platform/browserSignaling.ts  [NEW]
    - Pure browser WebSocket implementation of GFN signaling
    - Mirrors GfnSignalingClient from src/main/gfn/signaling.ts
    - Handles peer_info handshake, heartbeat, offer/answer forwarding,
      ICE candidate forwarding

  src/renderer/src/App.tsx
    - disconnected handler: ignores signaling disconnect if WebRTC is
      already connected (clientRef.current.isStreaming())
    - GFN server closes signaling WebSocket after offer/answer -- this
      is normal and must not be treated as stream end

  src/renderer/src/gfn/webrtcClient.ts
    - Added isStreaming() method
    - Added import for getPlatformApi
    - sendIceCandidate: falls back to getPlatformApi().sendIceCandidate
      on Android (window.openNow not available in WebView)
    - sendAnswer: same pattern

    - Calls setOrientation("sensor") when back to idle

Behavior:
  - Library, home, settings: free portrait+landscape (unchanged)
  - Session loading (queue/setup/connecting): also free (lock only on "streaming")
  - Streaming: forced landscape (either orientation sensor side)
  - After stop: back to free rotation

-- SESSION 5 (continued) --
Added region support for Android settings page.

Changes made:
  android/app/src/main/java/com/zortos/opennow/GfnPlugin.kt
    - getRegions now accepts token + streamingBaseUrl params
    - Calls /v2/zone on the streaming base URL with auth headers
    - Parses zones/data/regions array, extracts name and url fields
    - Falls back gracefully if parse fails (returns empty array)

  src/renderer/src/platform/api.ts
    - getRegions in buildCapacitorApi now extracts token and baseUrl
      from the input object and forwards them to the native plugin

  src/renderer/src/App.tsx
    - Both getRegions calls (startup restore + post-login) now pass
      providerStreamingBaseUrl alongside the token

Note: The actual response shape from /v2/zone is unknown until first
test. The Kotlin parser tries zones, data, and regions as top-level
keys, and name/zoneName/id for the display name, and url/streamingUrl/
serverUrl for the URL. Check logcat (GfnPlugin tag) if empty.

getLoginProviders now fetches live from:
  GET https://pcs.geforcenow.com/v1/serviceUrls
Parsed from gfnServiceInfo.gfnServiceEndpoints array.
Falls back to hardcoded NVIDIA default if fetch/parse fails.
Fields mapped: idpId, loginProviderCode->code, loginProviderDisplayName,
  streamingServiceUrl, loginProviderPriority->priority.
